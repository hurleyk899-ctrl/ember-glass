# Contributing to Ember Glass 🌿

Thanks for your interest in contributing! Here's how to get involved.

## 🔀 Branch Strategy

- `main` — production-ready code only. Protected branch.
- `dev` — active development. All PRs merge here first.
- Feature branches: `feature/your-feature-name`
- Bug fix branches: `fix/issue-description`

## 🛠 Development Workflow

1. Fork the repo and create a branch from `dev`
2. Make your changes with clear, atomic commits
3. Write or update tests as needed
4. Open a PR to `dev` — not `main`
5. Wait for review and approval

## 📝 Commit Message Format

Use conventional commits:

```
feat: add strain search filter
fix: dispensary map not loading on Android
docs: update API integration guide
chore: bump dependencies
```

## 🐛 Reporting Bugs

Use the **Bug Report** issue template. Include:
- Steps to reproduce
- Expected vs actual behavior
- Device/OS/app version
- Screenshots if applicable

## 💡 Suggesting Features

Use the **Feature Request** issue template. Describe:
- The problem you're solving
- Your proposed solution
- Any alternatives considered

## 🌱 Strain & Dispensary Data

For data-related contributions (new strain info, dispensary details), use the **Data Contribution** issue template.

*Be respectful, be helpful, be excellent to each other.*
